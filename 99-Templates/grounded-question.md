# Grounded question

**Question:**

**Topic / lecture:**

**Required behavior:**

1. Read the relevant lecture note in `10-Courses/Computer-Vision/lectures/`, the relevant concept notes in `20-Concepts/`, and the underlying PDF in `30-Sources/Computer-Vision/pdf/` if the notes aren't enough.

2. **Explain it clearly first.** Write the answer like you're teaching me — natural prose, normal markdown formatting (bold, italics, equations, lists where they actually help). Use diagrams (Mermaid) when they clarify something. The goal is that I understand the concept, not that I can audit every claim.

3. **Stay grounded.** Don't invent details that weren't in the source. If you're going outside the lecture, say so inline with a short note like *(not in lecture, but worth knowing)* — keep it conversational, not bureaucratic.

4. **At the end, three short sections:**

   - **The source's exact framing** — one short paragraph capturing how the lecture actually phrases this. Italicized, no blockquote. Just the lecture's wording so I match it on the exam.
   - **Remember this** — the one-line atomic fact for flashcards.
   - **Beyond the lecture** — anything you added that wasn't in the source. Skip if you didn't add anything.

5. No `> blockquote` formatting in the body. Use bold, italics, lists, equations, diagrams — but write in flowing prose, not in stacked claim-quote-citation triples.
