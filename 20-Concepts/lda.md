# Latent Dirichlet Allocation (LDA)

**LDA** (Blei, Ng, Jordan 2003) is a **probabilistic generative model** that represents each document as a **mixture over latent topics**, and each topic as a **distribution over words**. Where [[latent-semantic-analysis|LSA]] decomposes the term-document matrix with linear algebra (SVD), LDA decomposes it with **Bayesian inference** — same goal (uncover hidden semantic axes), different machinery.

The blueprint flags LDA as **medium weight**: mock Q8 (definition), Quiz III Q5, Q17 (and B mirrors) test the generative story and the role of the Dirichlet prior. The full LDA equations are on the formula sheet — what the exam targets is the *meaning* of the symbols.

## The two-level generative story

LDA assumes every document was generated by the following recipe:

1. For each document $d$, draw a **topic proportions vector** $\theta_d \sim \mathrm{Dir}(\alpha)$ — a length-$K$ probability distribution saying "this document is 60% topic 1, 30% topic 4, …".
2. For each topic $k$, draw a **word distribution** $\varphi_k \sim \mathrm{Dir}(\beta)$ — a length-$|V|$ distribution saying "topic $k$ is mostly the words {neuron, gradient, loss, …}".
3. For each word position in $d$:
   a. Sample a topic $z \sim \theta_d$.
   b. Sample the actual word $w \sim \varphi_z$.

So a word is generated by **first picking which topic this position is about, then picking a word from that topic**. The same word can appear under multiple topics with different probabilities ("bank" appears in finance and geography topics).

## Formulas (on the sheet)

| Quantity | Definition |
|---|---|
| Dirichlet prior | $\mathrm{Dir}(\alpha_1, \ldots, \alpha_K)$ over the $K$-simplex |
| Expected proportion of topic $k$ | $\mathbb{E}[\theta_k] = \alpha_k / \sum_j \alpha_j$ |
| Document-topic distribution | $\theta_d \sim \mathrm{Dir}(\alpha)$ |
| Topic-word distribution | $\varphi_k \sim \mathrm{Dir}(\beta)$ |
| Word generation | $z \sim \theta_d$, $w \sim \varphi_z$ |
| Posterior topic proportions | $\theta_k \propto n_k + \alpha_k$ (where $n_k$ counts how often $z = k$ in the document) |
| Posterior topic-word | $\varphi_w \propto n_w + \beta_w$ |

## What the Dirichlet prior controls (the conceptual MCQ trap)

The Dirichlet prior $\mathrm{Dir}(\alpha)$ is the **distribution over distributions** — it generates the per-document topic mixtures. The single hyperparameter $\alpha$ controls **sparsity**:

- **$\alpha < 1$**: sparse — most documents are concentrated on a *few* topics. Realistic for short documents.
- **$\alpha = 1$**: uniform over the simplex — every topic mixture is equally likely.
- **$\alpha > 1$**: dense — most documents spread mass *evenly* across all topics. Unrealistic for natural text.

> "Dirichlet prior controls **sparsity of topic proportions**." (blueprint, "What the formula sheet does NOT provide")

The same logic applies to $\beta$ for the word distribution within each topic — small $\beta$ → topics concentrated on few words.

## Inference (the bit not on the sheet)

LDA's posterior $P(\theta, \varphi, z \mid w, \alpha, \beta)$ has no closed form. In practice, parameters are estimated by:

- **Collapsed Gibbs sampling** — iteratively resample each word's topic assignment $z_i$ given all other assignments, using the posterior counts $\theta_k \propto n_k + \alpha_k$, $\varphi_w \propto n_w + \beta_w$.
- **Variational inference** — approximate the posterior with a tractable family (used in `gensim.models.LdaModel`).

The exam doesn't test the inference algorithm — it tests the *generative model* and the *meaning of the prior*.

## Contrast with LSA

| | [[latent-semantic-analysis|LSA]] | LDA |
|---|---|---|
| Type | Deterministic, linear-algebraic | Probabilistic, generative |
| Decomposition | $X = U \Sigma V^\top$ (truncated SVD) | $P(w, z, \theta, \varphi)$ via Dirichlet priors |
| "Topic" axes | Singular vectors (can have negative weights) | Probability distributions over words (non-negative, sum to 1) |
| New documents | Fold in via $V_k^\top X_{\text{new}}$ | Run inference to estimate $\theta_{\text{new}}$ |
| Output interpretability | Latent dimensions are abstract directions | Topics are probability distributions — directly readable as "topic 3 = {gene, protein, dna, …}" |

LSA emphasizes geometry; LDA emphasizes a generative story. Both **decompose document-word information into low-dim latent structure** — the connection that puts them in the same cluster as Word2Vec.

## Exam framing

| Question | Answer |
|---|---|
| What does LDA represent each document as? | A **mixture of latent topics** — a probability distribution over $K$ topics (mock Q8) |
| What does each topic represent? | A **probability distribution over the vocabulary** — words ranked by their probability under that topic |
| What does the Dirichlet prior control? | The **sparsity of topic proportions** — small $\alpha$ → documents concentrate on few topics; large $\alpha$ → topics spread evenly (Quiz III Q17) |
| Generative process for a single word? | (1) sample topic $z \sim \theta_d$, (2) sample word $w \sim \varphi_z$ (Quiz III Q5) |
| LDA vs LSA — one-line distinction? | LDA is **probabilistic and generative**; LSA is **linear and deterministic**. Both find latent low-dim structure in the term-document matrix |
| What does $\mathbb{E}[\theta_k] = \alpha_k / \sum_j \alpha_j$ tell you? | The **expected proportion of topic $k$** before seeing any data — the prior's contribution before evidence updates it |
