# Turing test (imitation test)

Alan Turing's 1950 proposal: evaluate machine intelligence through **observable behaviour**, not internal mechanism. If a machine's linguistic behaviour is indistinguishable from a human's, it should be considered intelligent.

This **reframed language** from an internal structure to an observable performance — the same conceptual move that enabled [[eliza|ELIZA]] and modern statistical NLP.

## The two-fold legacy

**Useful:** It defined a tractable evaluation criterion when no one knew how to model "understanding" formally. Behaviourally testable systems became possible.

**Limitation:** It rewards **imitation over understanding** (Quiz I Q22). A system that exploits conversational conventions ([[eliza]]) can pass without comprehension. Behavioural success therefore **can be misleading** (Quiz I Q24) — fluency does not guarantee semantic understanding.

## Modern echo

The contemporary concern that "echoes the ELIZA–Turing discussion about apparent understanding" is **hallucinated responses** (Quiz I Q23) — LLMs produce confident, fluent text that is nonetheless ungrounded.

## Exam framing

- Turing's test evaluates intelligence through **observable behaviour** (Quiz I Q21)
- A limitation is that it **may reward imitation over understanding** (Quiz I Q22)
- The central distinction this drives is **fluency vs understanding** (Quiz I Q9)
