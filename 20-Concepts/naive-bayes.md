# Naïve Bayes

A **probabilistic, generative** classifier based on [[bayes-formula|Bayes' formula]]. Models how documents are generated by each class rather than directly learning a decision boundary. The "naïve" refers to a strong simplifying assumption: **conditional independence of features given the label**.

## The classification rule (on the formula sheet)

$$P(y \mid x_1, \ldots, x_n) \propto P(y) \prod_i P(x_i \mid y)$$

Pick the class with the highest score:
$$\hat{y} = \arg\max_y \; P(y) \prod_i P(x_i \mid y)$$

The full Bayes derivation: posterior ∝ prior × likelihood. The "naïve" simplification turns the joint $P(x_1,\ldots,x_n|y)$ into a product, making the model tractable.

## The key assumption

> **Features are independent given the class label.**

(Mock Q21 answer; Quiz II.M3 Q12: "It assumes conditional independence of words.")

This is **false in natural language** (words depend strongly on neighbours), but the classifier often works well anyway — a famous mismatch between modelling assumption and empirical performance.

## Application to text

Documents are represented as [[bag-of-words]]. Each word contributes **independent evidence** toward a class. Each class defines a **probability distribution over the vocabulary**, and documents are modelled as sequences of word draws. What the model learns: **which words provide evidence for each class** — the model is interpretable (you can inspect which words push spam vs not-spam).

## Practical refinements

| Refinement            | Why                                                                                                                      |                                                              |                                                                                                                                                                 |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Log-probabilities** | Multiplying many small probabilities causes numerical underflow. Logs turn products into sums and stabilize computation. |                                                              |                                                                                                                                                                 |
| **[[laplace-smoothing | Laplace smoothing]]**                                                                                                    | If a word never appears in training data for a class, $P(x_i | y)=0$ would zero out the entire posterior. Smoothing prevents one unseen word from discarding a document. The smoothing constant is the only NB hyperparameter. |
|                       |                                                                                                                          |                                                              |                                                                                                                                                                 |

## Variants in scikit-learn

| Variant | When to use |
|---|---|
| `MultinomialNB` | Word counts (BoW) — standard for text |
| `BernoulliNB` | Binary BoW (presence/absence) |
| `GaussianNB` | Continuous features assumed Gaussian per class |

The choice depends on the **probability distribution** assumed for $P(x_i|y)$.

## Strengths and limitations

**Strengths**
- **Fast, simple, robust**
- Works well with **small datasets**
- Strong baseline for text classification
- Interpretable — you can see which words push a document toward each class

**Limitations**
- Independence assumption ignores **word order and syntax**
- Does not capture **semantic meaning**
- Saturates in performance on more complex tasks

## Worked exam shapes

> "Which assumption is made in Naïve Bayes classifiers?" → **Features are independent given the class label** (mock Q21).

> "Which statement about NB is correct?" → **It assumes conditional independence of words** (Quiz II.M3 Q12).

> "Which is NOT correct?" → "NB directly models $p(y|x)$ using a softmax decision rule" — NB is generative, parameterized via $p(x|y)$ and $p(y)$ (Quiz II Q18).

## Canonical code skeleton (sklearn)

The notebook uses scikit-learn's `MultinomialNB` for SMS spam classification:

```python
import pandas as pd
from collections import Counter
from sklearn.model_selection import train_test_split
from sklearn.naive_bayes import MultinomialNB
from sklearn import metrics

# 1. Load and binarize labels
sms = pd.read_csv(url, sep="\t", header=None, names=["label", "text"])
sms["label"] = sms["label"].map({"ham": 0, "spam": 1})

# 2. BoW representation (build vocab + count matrix)
vocab = sorted({w for text in sms["text"] for w in text.lower().split()})
def to_bow(text):
    counts = Counter(text.lower().split())
    return [counts[w] for w in vocab]

X = [to_bow(t) for t in sms["text"]]
y = sms["label"].values

# 3. Train/test split + fit
X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=42)

clf = MultinomialNB()  # alpha = 1.0 (Laplace smoothing) by default
clf.fit(X_train, y_train)

# 4. Predict + evaluate
y_pred = clf.predict(X_test)
print(metrics.classification_report(y_test, y_pred))
```
